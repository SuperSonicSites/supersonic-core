# Design Reference Library

Curated examples of what good looks like, per industry and per section
archetype. The `design-critic` skill anchors its scoring against these —
taste is pattern-matching against strong examples, so this library IS the
critic's taste. Curation is a human job: the site owner adds entries; agents
only read them.

## Entry format

One markdown file per case study: `<industry>-<name>.md` with:

- **Context** — industry, brand temperature, what the business sells
- **Screenshots** — committed under `docs/design-references/assets/`
  (own work or licensed only; for external sites, link + describe, never copy)
- **What works** — specific, mechanical observations (type scale jumps,
  band rhythm sequence, where photography carries emotion)
- **What to steal** — directives a layout-architect could apply verbatim
- **Personality mapping** — which styles/*.json personality this supports

## Seeding rule

Every completed engagement that passes the design critic adds its
before/after as a case study. The library compounds: each client makes the
next critique sharper.
