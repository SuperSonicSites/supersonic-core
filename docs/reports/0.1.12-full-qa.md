## The one to actually fix — Medium (1)

**section-card-row — heading-order skip (H3 with no H2)**
`patterns/section-card-row.php` — only headings are three H3 card titles; no H2 section
heading. Every sibling card/grid pattern uses H2→H3. Creates a heading-level skip
(WCAG 1.3.1) + SEO hierarchy issue. **Fix:** add an editable H2 (fontSize heading-2) as
the first child; keep card titles at H3.

## Systemic themes (the high-leverage cleanups)

These repeat across many patterns — fixing each is one mechanical pass:

1. **Inner-group `contentSize` not a token (860px / 920px)** — 9 patterns use an arbitrary
   inner width instead of the 760px `narrowWidth` token: cta-band, section-feature-grid,
   hero-centered, section-page-intro, section-process-steps, section-pricing-cards,
   section-media-feature (860px), and section-testimonial (920px). Renders fine; pure token
   hygiene. **Fix:** set each to `760px` (or drop to inherit 1440px). *(low/nit)*
2. **Media slots are styled-group "Replace with image" placeholders, not real `core/image`** —
   section-image-text, section-media-feature, section-text-image, contact-map-info. Editors
   get no native upload/alt-text controls. **Decision needed:** ship real `core/image`
   placeholders library-wide, or document the convention. *(low/nit)*
3. **Hardcoded `minHeight` px (420/460/520)** on those same media placeholders — no token
   exists. Resolves itself if media slots become real images. *(low/nit)*
4. **Placeholder `href="#"` + generic labels** — hero-simple, hero-centered, section-text-image,
   section-service-cards. Intentional starter copy; replace as pages are built. *(nit)*
5. **Redundant `layout:constrained` on inner card groups (no contentSize)** — contact-simple,
   section-comparison, section-testimonial-grid. Dead metadata; harmless. *(nit)*
6. **Quote semantics** — section-testimonial and section-testimonial-grid author quotes as
   styled paragraphs; `core/quote`+`cite` would be more semantic. *(nit)*
