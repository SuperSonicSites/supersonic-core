## Findings

### Blocker: `index.html` still injects a default H1 placeholder

The live staging homepage displays `Supersonic Site Theme` as an H1 from the fallback `templates/index.html`.

This conflicts with the established rule that H1s must come from AI-built page layouts or approved hero/intro patterns, not from a generic default template.

Recommended fix: make `templates/index.html` layout-neutral, matching the direction already taken for `page.html`.

### Blocker: mobile navigation overlay aligns links too far right

The mobile menu opens successfully, but the links hug the right edge of the overlay. This is visually weak and risks poor tap ergonomics.

Recommended fix: add scoped mobile-overlay CSS inside `assets/css/navigation.css` so the responsive container content has intentional spacing and left or centered alignment.

### Issue: footer site title wraps awkwardly with long site names/domains

The footer uses `core/site-title`. With the staging domain, the footer title wraps awkwardly on desktop and mobile.

Recommended fix: either make the footer identity text more defensive with scoped CSS or adjust the footer pattern so long site names do not dominate the footer.

### QA blocker: Rank Math FAQ block is not visible in the block registry

The registered FAQ pattern uses `rank-math/faq-block`, but staging did not expose a Rank Math/FAQ block type through the block registry.

Recommended fix: confirm Rank Math SEO is active and the Schema module / FAQ block is enabled before FAQ pattern QA.

### QA interference: chat widget appears in screenshots

A floating chat widget appears over the visual QA area on desktop, tablet, and mobile screenshots.

Recommended fix: disable the chat widget on staging during section-level screenshot QA, or document it as an intentional third-party overlay when reviewing affected screenshots.
