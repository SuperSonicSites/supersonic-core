# 0.1.31 Media Split Pattern QA

## Scope

Visual QA for the two Supersonic Media split patterns updated in theme 0.1.31:

- `supersonic-site-theme/section-text-image`
- `supersonic-site-theme/section-image-text`

## Contract

```text
Pattern: Section: Text + Image; Section: Image + Text
Category: supersonic-media
Selected block: outer core/group section; inner core/columns; core/image media slot
Promised controls: image replacement/alt/crop, section spacing/background, text column vertical alignment top/center/bottom, editable heading/body, editable buttons
Owning block for each control: image block owns media; outer group owns section rhythm/background; text column owns vertical alignment; text blocks own content/typography; buttons block and individual buttons own CTA layout, labels, URLs, and colors
Expected proof: staging version check, desktop/tablet/mobile section screenshots, pattern proof visibility/overflow/console checks
Manual-only gaps: wp-admin editor click-through for changing vertical alignment top/center/bottom was not performed in this run
```

## Proof Summary

- Static proof: `npm run validate` passed before release; `npm run pattern:registry:check` passed after QA evidence updates.
- Staging proof: `npm run rest:certify` confirmed WordPress 7.0, theme 0.1.31, plugin 0.1.8, and both registered patterns.
- Visual proof: desktop, tablet, and mobile screenshots captured for both patterns with selector `main .wp-block-group.alignfull:has(.supersonic-media-split)`.
- Interaction proof: not applicable; these patterns have no stateful UI.
- Editor-control proof: source structure uses native image, text, column, button, and section blocks; manual wp-admin alignment click-through remains a manual-only gap.
- Manual-only gaps: direct editor control testing for top/center/bottom vertical alignment was not performed.

## Review Notes

- Desktop preserves the intended side-by-side variants: text-left/image-right and image-left/text-right.
- Tablet and mobile stack the image above text for both variants.
- CTA buttons remain visible at all reviewed breakpoints.
- Image placeholders retain native image-block ownership and render with consistent cropping.
- Automated proof found one transient mobile console resource error on the first `section-image-text` run. A resource trace pointed to external analytics/consent requests, not pattern assets, and the rerun passed with no console or page errors.
