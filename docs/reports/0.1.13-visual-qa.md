## 7. Remaining Risks / Verify Visually

- **Screenshot coverage is the biggest gap.** Many patterns were verified by source only because the capture pipeline grabbed the default staging homepage instead of the pattern. Spacing, overflow, mobile stacking, and contrast for those patterns are **not** visually confirmed. Recommend creating per-pattern QA pages (`qa-pattern-[slug]`, dry-run + approval) and re-capturing before visual sign-off.
- **Placeholder `href="#"` and generic labels** across header, footer, hero, pricing, service-cards, cta-split, and contact-map-info must be wired before any page using these patterns goes live.
- **Hardcoded media `minHeight` (420/460/520px)** and the **contentSize "760px" sweep** remain open token-consistency items.
- **Site Identity:** header/footer logo gaps will persist on staging until a logo is uploaded or a `site-title` fallback is added.
