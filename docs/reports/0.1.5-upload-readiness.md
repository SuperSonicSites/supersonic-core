4. Confirm theme version `0.1.5` and plugin version `0.1.5`.
5. Confirm the expected Supersonic pattern categories are visible.
6. Confirm core/default/remote patterns remain absent.
7. Confirm Rank Math SEO and the Schema module are active before FAQ pattern QA.
8. Review one pattern or small pattern group at a time on staging-only QA pages.
9. Capture desktop, tablet, and mobile screenshots for each reviewed pattern section.
10. Run token editability checks from `QA_CHECKLIST.md`.

## Token Editability Scope

For the full `0.1.5` batch:

- do a deeper token editability pass on one representative pattern in each category
- spot-check the remaining patterns for text, link, media, color, spacing, typography, and block-validity controls
- undo test edits or discard the QA page after review

## Known Notes

- The FAQ pattern uses the approved `rank-math/faq-block`; the theme does not generate FAQ schema itself.
- The contact form pattern uses a native shortcode mount and requires an approved form implementation before production use.
- Map sections use editable placeholders until an approved map embed/image approach is selected.
- Generated zip files are ignored by Git and reproducible from source.
