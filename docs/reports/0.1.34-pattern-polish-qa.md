# 0.1.34 Pattern Polish Re-QA

**Date:** 2026-06-02
**Theme version:** 0.1.34
**Staging:** https://staging.yukonrealestateconnection.ca/
**Reviewer:** Deterministic Playwright probe (computed layout) + visual spot-check

## Scope

Re-QA of the multi-up responsive fix after a mobile regression shipped in 0.1.33.

- **Intent (M1, approved):** multi-up column bands hold horizontal at tablet (768px) but still stack on phones.
- **0.1.33 regression:** `is-not-stacked-on-mobile` also forces `flex-wrap: nowrap`, so the `flex-basis:100%` re-stack rule at `<=600px` could not wrap the columns — they crammed onto one row at 390px (pricing cards wrapped one character per line) instead of stacking.
- **0.1.34 fix:** add `flex-wrap: wrap !important` at `<=600px` so the full-width columns wrap to a single column on phones (`assets/css/patterns.css`).

## Method

For each affected pattern, a temporary `qa-pattern-*` staging page was created, probed with Playwright at 768px and 390px (computed bounding-box layout, computed `flex-wrap`, and document overflow), then trashed. Pass criteria: tablet `horizontal`, mobile `stacked`, no horizontal overflow, no console/page errors.
